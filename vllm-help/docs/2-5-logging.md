# 2.5 日志与调试

> 返回 [README 总览](../README.md)

配置 API 服务器的日志记录行为，包括日志级别、访问日志、请求/输出记录等。合理的日志配置对于生产环境的排障和监控至关重要。

---

## `--uvicorn-log-level`

| 属性 | 值 |
|------|-----|
| **说明** | 设置 Uvicorn（ASGI 服务器）的日志级别。日志级别从低到高依次为：`trace`（最详细）> `debug` > `info` > `warning` > `error` > `critical`（仅致命错误）。级别越低输出越多，对性能的影响也越大 |
| **类型** | enum |
| **可选值** | `debug` / `info` / `warning` / `error` / `critical` / `trace` |
| **默认值** | `info` |
| **暴露建议** | 平台管理 |

**示例：**
```bash
vllm serve --uvicorn-log-level warning
```
> 仅记录 warning 及以上级别的日志。适用于生产环境，减少日志量，只关注潜在问题和错误。

---

## `--disable-uvicorn-access-log`

| 属性 | 值 |
|------|-----|
| **说明** | 禁用 Uvicorn 的 HTTP 访问日志。访问日志记录每个 HTTP 请求的详细信息（路径、状态码、耗时等）。在高并发场景下，大量访问日志可能影响性能并占用磁盘空间 |
| **类型** | bool |
| **默认值** | `False` |
| **暴露建议** | 平台管理 |

**示例：**
```bash
vllm serve --disable-uvicorn-access-log
```
> 关闭 HTTP 访问日志记录。在高流量生产环境中，可以配合外部日志采集系统（如 Nginx access log）替代 Uvicorn 自身的访问日志。

---

## `--log-config-file`

| 属性 | 值 |
|------|-----|
| **说明** | 指定自定义的日志配置 JSON 文件路径。通过配置文件可以精细控制日志格式、输出目标（文件/终端/远程）、不同模块的日志级别等 |
| **类型** | string |
| **默认值** | None |
| **暴露建议** | 平台管理 |

**示例：**
```bash
vllm serve --log-config-file /etc/vllm/log-config.json
```
> 使用自定义日志配置文件。可以配置将日志输出到文件、设置日志轮转策略，或将不同级别的日志发送到不同的目标。

---

## `--log-error-stack`

| 属性 | 值 |
|------|-----|
| **说明** | 启用后，在日志中记录错误的完整堆栈信息。有助于定位异常的根本原因，但会增加日志体积 |
| **类型** | bool |
| **默认值** | `False` |
| **暴露建议** | 平台管理 |

**示例：**
```bash
vllm serve --log-error-stack
```
> 在错误日志中包含完整的 Python 堆栈跟踪信息。排查生产环境中偶发的模型推理错误或请求处理异常时非常有用。

---

## `--max-log-len`

| 属性 | 值 |
|------|-----|
| **说明** | 设置日志中打印的最大 prompt 字符数。当启用请求日志时，过长的 prompt 会被截断到此长度。防止超长 prompt 导致日志文件过大或日志系统过载 |
| **类型** | int |
| **默认值** | None（不限制） |
| **暴露建议** | 平台管理 |

**示例：**
```bash
vllm serve --enable-log-requests --max-log-len 500
```
> 启用请求日志并将 prompt 内容截断为最多 500 个字符。在需要记录请求内容但又担心日志过大时使用。

---

## `--enable-log-requests`

| 属性 | 值 |
|------|-----|
| **说明** | 启用请求级别的日志记录。开启后会记录每个 API 请求的详细信息，包括 prompt 内容、请求参数等。对调试和审计非常有用，但会增加日志量和轻微的性能开销 |
| **类型** | bool |
| **默认值** | `False` |
| **暴露建议** | 平台管理 |

**示例：**
```bash
vllm serve --enable-log-requests
```
> 记录所有 API 请求的详细信息。适用于开发调试阶段或需要审计请求内容的场景。

---

## `--enable-log-outputs`

| 属性 | 值 |
|------|-----|
| **说明** | 启用模型输出内容的日志记录。需要先启用 `--enable-log-requests` 才能生效。开启后会在日志中记录模型生成的完整输出文本。注意此选项会显著增加日志体积 |
| **类型** | bool |
| **默认值** | `False` |
| **暴露建议** | 平台管理 |

**示例：**
```bash
vllm serve --enable-log-requests --enable-log-outputs
```
> 同时记录请求的输入和模型的输出内容。适用于需要完整审计对话记录的合规场景，或排查模型输出质量问题。
