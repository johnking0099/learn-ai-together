# 12.1 通用选项

> 返回 [README 总览](../README.md)

vLLM 服务的全局通用配置选项，包括配置文件加载、API 服务进程管理、日志控制等。

---

## `--config`

| 属性 | 值 |
|------|-----|
| **说明** | YAML 配置文件路径。可以将所有命令行参数写入一个 YAML 文件中统一管理，避免命令行过长。配置文件中的参数名与命令行参数一致（去掉 `--` 前缀）。命令行参数的优先级高于配置文件 |
| **类型** | string |
| **默认值** | None |
| **暴露建议** | 平台决定 |

**示例：**
```bash
vllm serve --config /etc/vllm/serving-config.yaml
```
> 从指定的 YAML 配置文件加载所有参数，适合生产环境下的标准化部署管理。

---

## `--api-server-count` / `-asc`

| 属性 | 值 |
|------|-----|
| **说明** | API 服务进程数量。设置大于 1 时会启动多个 API 服务进程来处理请求，可以提升 API 层面的并发处理能力。适用于 API 层成为瓶颈的高并发场景 |
| **类型** | int |
| **默认值** | 1 |
| **暴露建议** | 平台决定 |

**示例：**
```bash
vllm serve Qwen/Qwen3-0.6B --api-server-count 4
```
> 启动 4 个 API 服务进程以提升 HTTP 请求的并发处理能力。

---

## `--disable-log-stats`

| 属性 | 值 |
|------|-----|
| **说明** | 禁用引擎统计日志的输出。默认情况下 vLLM 会定期记录吞吐量、延迟、队列长度等统计信息。在对日志量有严格要求的生产环境中，可以通过此选项关闭统计日志以减少 I/O 开销 |
| **类型** | bool |
| **默认值** | `False` |
| **暴露建议** | 高级配置 |

**示例：**
```bash
vllm serve Qwen/Qwen3-0.6B --disable-log-stats
```
> 关闭引擎统计日志输出，减少生产环境中的日志量。

---

## `--enable-log-requests`

| 属性 | 值 |
|------|-----|
| **说明** | 启用请求级别的日志记录。开启后会记录每个 API 请求的详细信息，包括请求 ID、输入长度、输出长度、处理时间等。对调试和审计非常有用，但在高并发场景下会产生大量日志 |
| **类型** | bool |
| **默认值** | `False` |
| **暴露建议** | 高级配置 |

**示例：**
```bash
vllm serve Qwen/Qwen3-0.6B --enable-log-requests
```
> 启用请求日志，记录每个推理请求的详细信息，便于问题排查和使用量审计。

---

## `--aggregate-engine-logging`

| 属性 | 值 |
|------|-----|
| **说明** | 在数据并行部署场景下，将多个引擎实例的日志聚合输出。避免多个并行实例各自输出独立的统计日志导致日志混乱，改为统一汇总后输出 |
| **类型** | bool |
| **默认值** | `False` |
| **暴露建议** | 高级配置 |

**示例：**
```bash
vllm serve Qwen/Qwen3-0.6B -dp 4 --aggregate-engine-logging
```
> 在 4 路数据并行部署中聚合所有引擎实例的日志，输出统一的汇总统计信息。

---

## `--headless`

| 属性 | 值 |
|------|-----|
| **说明** | 以无头模式启动 vLLM。在此模式下不启动 HTTP API 服务器，仅启动推理引擎。适用于多节点数据并行场景中的非主节点，这些节点只需要运行推理引擎而不需要对外暴露 API 端点 |
| **类型** | bool |
| **默认值** | `False` |
| **暴露建议** | 平台决定 |

**示例：**
```bash
vllm serve Qwen/Qwen3-0.6B --headless --data-parallel-address 10.0.0.1
```
> 以无头模式启动次要节点的引擎，仅运行推理计算，API 请求由主节点的 HTTP 服务接收并分发。
