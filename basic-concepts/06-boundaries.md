# 06 - 不需要深入的领域：划清边界

知道"不需要学什么"和"需要学什么"同样重要。以下是 AI 产品经理可以放心交给工程师的领域——但对每个领域，我们也标注了"你至少需要知道到什么程度"。

---

## CUDA Kernel 编写

**不需要**：学习 CUDA 编程、理解 GPU 的线程模型（warp、block、grid）、编写或优化 kernel 代码。

**但你需要知道**：
- CUDA 是 NVIDIA GPU 的编程接口，几乎所有深度学习框架都依赖它
- 这就是为什么"NVIDIA GPU 垄断 AI 市场"——生态锁定
- 当工程师说"需要写 custom kernel"时，意味着这是一项有门槛的工作，需要排期

---

## Attention 的数学推导

**不需要**：理解 Q、K、V 矩阵乘法的细节、softmax 归一化的数学原理、FlashAttention 的算法优化。

**但你需要知道**：
- Attention（注意力机制）是 Transformer 模型的核心组件
- 它让模型能"关注"输入中最相关的部分——这就是为什么 LLM 能理解上下文
- **上下文长度的计算成本与长度的平方成正比**——这解释了为什么 128K 上下文比 4K 贵得多，不只是 32 倍
- FlashAttention 等优化技术能显著降低这个成本，但不改变根本的 scaling 关系

---

## 自己训练/微调模型

**不需要**：掌握训练框架（PyTorch、DeepSpeed）的使用、设计训练数据集、调整 learning rate 等超参数、管理训练集群。

**但你需要知道**：

### 训练的基本流程
1. **预训练（Pre-training）**：用海量数据从零训练，成本极高（百万到千万美元级）
2. **微调（Fine-tuning）**：在预训练模型基础上用特定数据调整，成本可控（数千到数万美元）
3. **LoRA 微调**：更轻量的微调方式，只训练一小部分参数，成本低（数百到数千美元）

### 对产品决策的影响
- 当通用模型效果不够好时，微调是一个选项——但需要数据和工程投入
- LoRA 微调的门槛远低于全量微调，可以纳入产品功能（让用户微调自己的模型）
- 微调数据的质量远比数量重要——"500 条高质量数据 > 50,000 条噪声数据"

---

## 网络架构细节

**不需要**：理解 Transformer 的每一层（embedding、layer norm、FFN）的具体实现、不同架构变体（GPT vs BERT vs T5）的技术差异、模型架构搜索（NAS）。

**但你需要知道**：
- **Transformer** 是当前几乎所有大模型的基础架构——LLM、扩散模型、多模态模型都基于它
- 模型的"层数"和"隐藏维度"决定了参数量，参数量决定了显存需求
- **Decoder-only**（如 GPT、Llama）：主流 LLM 架构，逐个生成 token
- **Encoder-Decoder**（如 T5）：适合翻译等"输入→输出"任务，目前较少见

---

## 分布式训练技术

**不需要**：理解 FSDP、ZeRO、梯度累积、混合精度训练等分布式训练技术的实现细节。

**但你需要知道**：
- 大模型训练需要大量 GPU 协同工作，这是一个工程挑战
- 训练不稳定性（loss spike、NaN）是常见问题，可能导致训练中断和时间浪费
- 这就是为什么"训练一个大模型"需要专门的团队和基础设施
