# 05 - 图像生成模型的特殊知识

如果你的产品涉及图像生成（文生图、图生图、图像编辑），这一章覆盖与 LLM 截然不同的技术栈和产品考量。

---

## 扩散模型基础

当前主流的图像生成模型都基于**扩散模型（Diffusion Model）**。核心原理用一句话概括：**从纯噪声开始，一步步"去噪"，最终生成清晰的图像。**

**直觉类比**：像雕塑家从一块粗糙的石头开始，一刀一刀地雕刻出最终的作品。每一刀就是一个"step"。

### 用户可调的核心参数

| 参数 | 含义 | 对结果的影响 |
|------|------|-------------|
| **Steps**（步数） | 去噪的迭代次数 | 越多细节越丰富，但更慢。通常 20-50 步 |
| **CFG Scale** | 对文字提示的"服从程度" | 越高越严格遵循提示词，但过高会失真 |
| **Sampler** | 去噪算法 | 不同算法出图风格和速度不同（如 Euler、DPM++） |
| **Seed** | 随机种子 | 相同种子 + 相同参数 = 相同图片，方便复现 |
| **尺寸** | 输出图片的宽高 | 受模型训练尺寸限制，偏离太多效果变差 |

## 与 LLM 的关键差异

图像生成和 LLM 推理看起来都是"模型推理"，但技术特征差异很大：

| 维度 | LLM | 图像生成 |
|------|-----|---------|
| **输出方式** | 逐 token 流式输出 | 全部计算完一次性输出 |
| **延迟特征** | TTFT + 持续生成（秒级起） | 固定等待时间（数秒到数十秒） |
| **显存占用** | 模型 + KV Cache（动态增长） | 模型 + 中间图像（相对固定） |
| **并发模型** | 靠 KV Cache 管理实现高并发 | 主要靠 batch size 和多卡 |
| **用户交互** | 对话式，可中途停止 | 等待式，生成中难以中断 |
| **计费模型** | 按 token 数（输入+输出） | 按图片数或按分辨率/步数 |

**对产品意味着什么**：图像生成的用户等待体验完全不同——没有"逐字输出"的渐进感，用户只能等。因此进度指示（progress bar）和预期时间提示对体验至关重要。

## 工具链与社区生态

图像生成领域有活跃的开源社区和工具链：

| 工具 | 定位 | 特点 |
|------|------|------|
| **ComfyUI** | 节点式工作流编辑器 | 灵活、可组合、社区活跃，适合专业用户 |
| **A1111 (Stable Diffusion WebUI)** | 一体化 WebUI | 上手简单，功能全面，社区最大 |
| **Forge** | A1111 的优化分支 | 更低显存占用，更快速度 |

**对产品意味着什么**：图像生成领域的"平台级产品"需要与社区生态兼容——支持社区模型格式（SafeTensors）、工作流格式（ComfyUI JSON）等。

## 控制手段的产品化

让用户更精确地控制生成结果是图像生成产品的核心竞争力：

### ControlNet

通过额外的条件输入（边缘图、深度图、姿态图等）控制生成图像的结构。

**产品场景**：用户上传一张草图，生成保持相同构图的精美图像。

### LoRA（图像生成版）

与 LLM 的 LoRA 概念类似，但用于控制图像风格或角色一致性。

**产品场景**：用户训练自己的人物 LoRA，生成该人物的各种场景图片。

### IP-Adapter

用参考图片来控制生成图像的风格或内容元素。

**产品场景**：用户提供一张参考图，生成"风格相似但内容不同"的新图。

## 图像生成服务化的特殊挑战

把图像生成做成平台服务，比 LLM 服务化面临更多独特挑战：

| 挑战 | 说明 | 应对思路 |
|------|------|---------|
| **模型多样性** | 社区模型数以千计，格式和版本繁多 | 支持主流格式，建立模型审核流程 |
| **工作流复杂度** | 专业用户的工作流涉及多模型串联 | 提供工作流编排能力（参考 ComfyUI） |
| **内容安全** | 图像生成的 NSFW 风险远高于文本 | 必须有生成前后的安全过滤 |
| **结果不确定性** | 同样的提示词，不同随机种子结果完全不同 | 提供批量生成 + 挑选的体验 |
| **资源隔离** | 大图 / 高步数请求可能独占 GPU 很长时间 | 需要超时机制和队列管理 |

**核心产品决策**：图像生成平台是面向"专业创作者"（提供灵活性）还是"普通用户"（提供简单性）？这个定位决定了几乎所有产品设计选择。
